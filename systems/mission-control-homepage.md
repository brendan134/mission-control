# Mission Control Overview

## Purpose
Mission Control is the operator-facing control layer for the Leader By Design AI system.

It should make it easy to:
- understand current priorities
- access core operating rules
- navigate workflows and templates
- manage model usage
- find the right system files quickly

---

## Core Principle
Mission Control is a view over portable files.
The files remain the source of truth.

---

## What Mission Control Should Show

### 1. Current State
- current priorities
- active projects
- immediate next actions
- important blockers

### 2. Operator Guidance
- operator cheat sheet
- operating rules
- operating constraints
- escalation guidance

### 3. Planning
- weekly planning
- quarterly focus
- project review

### 4. Models
- current default model
- routing policy
- escalation rules
- cost-saving logic

### 5. Execution Assets
- workflows
- templates
- project files
- decision records

---

1. System Snapshot

10. System Docs

---

## Operator Outcome
A good Mission Control homepage should help the operator answer:
- What matters right now?
- What should I do next?
- Which file or workflow should I open?
- Which model should I use?
- Where should this information be saved?

---

## Design Rule
Mission Control should reduce friction, not add complexity.
If a section does not help the operator make decisions or take action, simplify it.
