## Cloudflare Tunnel Troubleshooting (April 26, 2026)

### Issue
- Mission Control returning Error 1033 / 530
- Cloudflare tunnel showing "status = down"
- Error: "control stream encountered a failure while serving"
- Later: "Unauthorized: Tunnel not found"

### Root Causes
1. **Tunnel credentials invalidated** - Cloudflare tunnel tokens can expire/become invalid
2. **PM2 process management issue** - Running cloudflared via PM2 caused Cloudflare to reject connections
3. **Missing ingress rule** - New tunnel needed public hostname route configured

### Solution Applied
1. **Recreated tunnel in Cloudflare Dashboard**
   - Deleted old "Mission-Control" tunnel
   - Created new tunnel with same name
   - Added CNAME DNS record (auto-created by Cloudflare)
   
2. **Configured public hostname route**
   - Hostname: `mc.brendanrogers.au`
   - Service: `http://127.0.0.1:3003` (NOT localhost - IPv4 issue)

3. **Used nohup instead of PM2**
   - PM2 caused tunnel connection rejection
   - Using `nohup /path/to/cloudflared tunnel run --token "..." > /tmp/cloudflared.log 2>&1 &` works

### Key Learnings
1. **Always use `127.0.0.1` not `localhost`** in tunnel ingress rules (IPv6 resolves to ::1 inside container)
2. **nohup > PM2** for cloudflared - PM2's process management causes Cloudflare to reject connections
3. **Token must match** - Tunnel ID in token must match the tunnel in Cloudflare Dashboard
4. **Public hostname route required** - Just creating tunnel isn't enough; must add ingress rule

### Startup Script
Location: `/data/.openclaw/start-tunnel.sh`
- Updated with new token (April 26, 2026)
- Uses nohup approach
- Must be manually triggered after container restart
