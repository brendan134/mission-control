# Task Capture and Prioritisation Workflow

## Purpose
Create a simple, repeatable way to capture, clarify, prioritise, and route work.

## Goal
Turn requests, ideas, and loose tasks into clear next actions.

## Step 1: Capture
Capture tasks from:
- chat
- meetings
- voice notes
- ideas
- email/messages
- project discussions

## Step 2: Clarify
For each captured item, define:
- what needs to happen
- desired outcome
- owner
- due date or timing
- first next action

If unclear, rewrite it until it is actionable.

## Step 3: Classify
Classify each item as one of:
- do now
- schedule
- delegate
- defer
- drop
- incubate

## Step 4: Prioritise
Use this priority system:
- P1 = critical and time-sensitive
- P2 = important but not urgent
- P3 = useful but lower value
- P4 = optional / someday

## Step 5: Route
Route each task into the correct place:
- weekly plan
- project file
- delegated list
- content workflow
- archive / dropped items

## Decision Rules
- If it supports a current strategic priority, elevate it.
- If someone else should own it, delegate it.
- If it is not actionable, clarify it before storing it.
- If it does not matter, drop it.
- If it matters later, defer it with a review point.

## Standard Task Format
- Task:
- Outcome:
- Priority:
- Owner:
- Due:
- Next action:
- Status:

## Operating Rule
No important task should remain only in chat.
